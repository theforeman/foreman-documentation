[id="hammer-repository-create"]
= repository create

[role="_abstract"]
Create a custom repository

*Usage*

----
# hammer repository create [OPTIONS]
----

*Options*

* `--ansible-collection-auth-token VALUE` {endash} The token key to use for authentication.
* `--ansible-collection-auth-url VALUE` {endash} The URL to receive a session token from, e.g. used with Automation Hub.
* `--ansible-collection-requirements VALUE` {endash} Contents of requirement yaml file to sync from URL
* `--ansible-collection-requirements-file VALUE` Location of the ansible collections requirements.yml file.
* `--arch VALUE` {endash} Architecture of content in the repository
* `--checksum-type VALUE` {endash} Checksum used for published repository contents. Supported types: sha256,
sha384, sha512
* `--content-type VALUE` {endash} Type of repository to create. View available types with "hammer repository
types"
* `--deb-architectures VALUE` {endash} Whitespace-separated list of architectures to be synced from deb-archive
* `--deb-components VALUE` {endash} Whitespace-separated list of repo components to be synced from deb-archive
* `--deb-releases VALUE` {endash} Whitespace-separated list of releases to be synced from deb-archive
* `--depth NUMBER` {endash} An option to specify how many ostree commits to traverse.
* `--description VALUE` {endash} Description of the repository
* `--docker-upstream-name VALUE` {endash} Name of the upstream docker repository
* `--download-concurrency NUMBER` {endash} Used to determine download concurrency of the repository in pulp3. Use value
less than 20. Defaults to 10
* `--download-policy ENUM` {endash} Download policy for deb, docker, file and yum repos (either `immediate` or
`on_demand`)
Possible value(s): `immediate`, `on_demand`
* `--exclude-refs LIST` {endash} A comma-separated list of tags to exclude during an ostree sync. The wildcards
*, ? are recognized. `exclude_refs` is evaluated after `include_refs`.
* `--exclude-tags LIST` {endash} Comma-separated list of tags to exclude when syncing a container image
repository. Default: any tag ending in “-source”
* `--excludes LIST` {endash} Python packages to exclude from the upstream URL, names separated by newline.
You may also specify versions, for example: django~=2.0.
* `--gpg-key-id NUMBER` {endash} Id of the gpg key that will be assigned to the new repository
* `--http-proxy VALUE` {endash} Name to search by
* `--http-proxy-id NUMBER` {endash} ID of a HTTP proxy
* `--http-proxy-policy ENUM` {endash} Policies for HTTP proxy for content sync
Possible value(s): `global_default_http_proxy`, `none`,
`use_selected_http_proxy`
* `--ignorable-content LIST` {endash} List of content units to ignore while syncing a yum repository. Must be subset
of srpm,treeinfo
* `--include-refs LIST` {endash} A comma-separated list of refs to include during an ostree sync. The wildcards
*, ? are recognized.
* `--include-tags LIST` {endash} Comma-separated list of tags to sync for a container image repository
* `--includes LIST` {endash} Python packages to include from the upstream URL, names separated by newline.
You may also specify versions, for example: django~=2.0. Leave empty to include
every package.
* `--keep-latest-packages NUMBER` {endash} The amount of latest versions of a package to keep on sync, includes
pre-releases if synced. Default 0 keeps all versions.
* `--label VALUE`
* `--metadata-expire NUMBER` {endash} Time to expire yum metadata in seconds. Only relevant for custom yum
repositories.
* `--mirroring-policy ENUM` {endash} Policy to set for mirroring content.  Must be one of additive.
Possible value(s): `additive`, `mirror_complete`, `mirror_content_only`
* `--name VALUE` {endash} Name of the repository
* `--organization VALUE` {endash} Organization name to search by
* `--organization-id NUMBER`
* `--organization-label VALUE` {endash} Organization label to search by
* `--organization-title VALUE` {endash} Organization title
* `--os-versions LIST` {endash} Identifies whether the repository should be unavailable on a client with a
non-matching OS version. Pass [] to make repo available for clients regardless
of OS version. Maximum length 1; allowed tags are: rhel-6, rhel-7, rhel-8,
rhel-9, rhel-10
* `--package-types LIST` {endash} Package types to sync for Python content, separated by comma. Leave empty to get
every package type. Package types are:
bdist_dmg,bdist_dumb,bdist_egg,bdist_msi,bdist_rpm,bdist_wheel,bdist_wininst,sdist.
* `--product VALUE` {endash} Product name to search by
* `--product-id NUMBER` {endash} Product the repository belongs to
* `--publish-via-http BOOLEAN` {endash} Publish Via HTTP
* `--retain-package-versions-count NUMBER` {endash} The maximum number of versions of each package to keep.
* `--ssl-ca-cert-id NUMBER` {endash} Identifier of the content credential containing the SSL CA Cert
* `--ssl-client-cert-id NUMBER` {endash} Identifier of the content credential containing the SSL Client Cert
* `--ssl-client-key-id NUMBER` {endash} Identifier of the content credential containing the SSL Client Key
* `--upstream-authentication-token VALUE` {endash} Upstream authentication token string for yum repositories.
* `--upstream-password VALUE` {endash} Password of the upstream repository user used for authentication
* `--upstream-username VALUE` {endash} Username of the upstream repository user used for authentication
* `--url VALUE` {endash} Repository source url
* `--verify-ssl-on-sync BOOLEAN` {endash} If true, Katello will verify the upstream url`s SSL certifcates are signed by a
trusted CA
* `-h`, `--help` {endash} Print help
