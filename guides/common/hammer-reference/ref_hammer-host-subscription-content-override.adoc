[id="hammer-host-subscription-content-override"]
= host subscription content-override

[role="_abstract"]
Override product content defaults

*Usage*

----
# hammer host subscription content-override [OPTIONS]
----

*Options*

* `--content-label VALUE` {endash} Label of the content
* `--enabled BOOLEAN` {endash} Set true to override to enabled; Set false to override to disabled.`
* `--force` {endash} Force the override. Required for overrides other than `enabled`
* `--full-result BOOLEAN` {endash} Whether or not to show all results
* `--host VALUE` {endash} Host name
* `--host-id VALUE` {endash} Id of the content host
* `--limit-to-env BOOLEAN` {endash} Limit actions to content in the host`s environment.
* `--order VALUE` {endash} Sort field and order, eg. `id DESC`
* `--override-name VALUE` {endash} Override parameter key or name.
To enable or disable a repo select `enabled`.
Default value: enabled
Default: "enabled"
* `--page NUMBER` {endash} Page number, starting at 1
* `--per-page NUMBER` {endash} Number of results per page to return
* `--remove` {endash} Remove a content override
* `--search VALUE` {endash} Search string
* `--sort-by VALUE` {endash} Field to sort the results on
* `--sort-order VALUE` {endash} How to order the sorted results (e.g. ASC for ascending)
* `--value VALUE` {endash} Override value. Note for repo enablement you can use a boolean value
* `-h`, `--help` {endash} Print help
