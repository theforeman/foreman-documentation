= ansible variables update

[role="_abstract"]
Updates Ansible variable

*Usage*

----
# hammer ansible variables update [OPTIONS]
----

*Options*

* `--ansible-role VALUE` {endash} Name to search by
* `--ansible-role-id NUMBER` {endash} Role ID
* `--avoid-duplicates BOOLEAN` {endash} Remove duplicate values (only array type)
* `--default-value VALUE` {endash} Default value of variable
* `--description VALUE` {endash} Description of variable
* `--hidden-value BOOLEAN` {endash} When enabled the parameter is hidden in the UI
* `--id VALUE`
* `--location VALUE` {endash} Set the current location context for the request
* `--location-id NUMBER` {endash} Set the current location context for the request
* `--location-title VALUE` {endash} Set the current location context for the request
* `--merge-default BOOLEAN` {endash} Include default value when merging all matching values
* `--merge-overrides BOOLEAN` {endash} Merge all matching values (only array/hash type)
* `--name VALUE` {endash} Name to search by
* `--new-name VALUE`
* `--organization VALUE` {endash} Set the current organization context for the request
* `--organization-id NUMBER` {endash} Set the current organization context for the request
* `--organization-title VALUE` {endash} Set the current organization context for the request
* `--override BOOLEAN` {endash} Whether to override variable or not
* `--override-value-order LIST` {endash} The order in which values are resolved
* `--validator-rule VALUE` {endash} Used to enforce certain values for the parameter values
* `--validator-type ENUM` {endash} Types of validation values
Possible value(s): `regexp`, `list`
* `--variable VALUE` {endash} Name of variable
* `--variable-type ENUM` {endash} Types of variable values
Possible value(s): `string`, `boolean`, `integer`, `real`, `array`, `hash`,
`yaml`, `json`
* `-h`, `--help` {endash} Print help
