= repository upload-content

[role="_abstract"]
This endpoint is primarily designed for UI interactions and uploading content into the repository. For API-based uploads, please use the 'content_uploads' endpoint instead.

*Usage*

----
# hammer repository upload-content [OPTIONS]
----

*Options*

* `--async` {endash} Do not wait for the task.
* `--content-type VALUE` {endash} The type of content unit to upload (srpm, file, etc.). View uploadable types
with "hammer repository types"
* `--fields LIST` {endash} Show specified fields or predefined field sets only. (See below)
* `--id NUMBER` {endash} Repository ID
* `--name VALUE` {endash} Repository name to search by
* `--organization VALUE` {endash} Organization name to search by
* `--organization-id NUMBER` {endash} Organization ID
* `--organization-label VALUE` {endash} Organization label to search by
* `--organization-title VALUE` {endash} Organization title
* `--ostree-repository-name VALUE` Name of OSTree repository in archive.
* `--path FILE` {endash} Upload file, directory of files, or glob of files as content for a repository.
Globs must be escaped by single or double quotes
* `--product VALUE` {endash} Product name to search by
* `--product-id NUMBER` {endash} Product numeric identifier
* `-h`, `--help` {endash} Print help
