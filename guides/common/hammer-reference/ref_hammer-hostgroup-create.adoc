[id="hammer-hostgroup-create"]
= hostgroup create

[role="_abstract"]
Create a host group

*Usage*

----
# hammer hostgroup create [OPTIONS]
----

*Options*

* `--ansible-role-ids LIST` {endash} IDs of associated ansible roles
* `--ansible-roles LIST`
* `--architecture VALUE` {endash} Architecture name
* `--architecture-id NUMBER` {endash} Architecture ID
* `--ask-root-password BOOLEAN`
* `--compute-profile VALUE` {endash} Compute profile name
* `--compute-profile-id NUMBER` {endash} Compute profile ID
* `--compute-resource VALUE` {endash} Compute resource name
* `--compute-resource-id NUMBER` {endash} Compute resource ID
* `--config-group-ids LIST` {endash} IDs of associated config groups
* `--config-groups LIST`
* `--content-source VALUE` {endash} Content Source name
* `--content-source-id NUMBER` {endash} Content source ID
* `--content-view VALUE` {endash} Name to search by
* `--content-view-id NUMBER` {endash} Content view ID
* `--description VALUE` {endash} Host group description
* `--domain VALUE` {endash} Domain name
* `--domain-id NUMBER` {endash} Domain ID
* `--group-parameters-attributes SCHEMA` Array of parameters
* `--kickstart-repository VALUE` {endash} Kickstart repository name
* `--kickstart-repository-id NUMBER` {endash} Kickstart repository ID
* `--lifecycle-environment VALUE` {endash} Name to search by
* `--lifecycle-environment-id NUMBER` {endash} Lifecycle environment ID
* `--location VALUE` {endash} Set the current location context for the request
* `--location-id NUMBER` {endash} Set the current location context for the request
* `--location-ids LIST` {endash} REPLACE locations with given ids
* `--location-title VALUE` {endash} Set the current location context for the request
* `--location-titles LIST`
* `--locations LIST`
* `--medium VALUE` {endash} Medium name
* `--medium-id NUMBER` {endash} Media ID
* `--name VALUE` {endash} Name of the host group
* `--openscap-proxy-id NUMBER` {endash} ID of OpenSCAP Capsule
* `--operatingsystem VALUE` {endash} Operating system title
* `--operatingsystem-id NUMBER` {endash} Operating system ID
* `--organization VALUE` {endash} Set the current organization context for the request
* `--organization-id NUMBER` {endash} Set the current organization context for the request
* `--organization-ids LIST` {endash} REPLACE organizations with given ids.
* `--organization-title VALUE` {endash} Set the current organization context for the request
* `--organization-titles LIST`
* `--organizations LIST`
* `--parent VALUE` {endash} Name of parent hostgroup
* `--parent-id NUMBER` {endash} Parent ID of the host group
* `--parent-title VALUE` {endash} Title of parent hostgroup
* `--partition-table VALUE` {endash} Partition table name
* `--partition-table-id NUMBER` {endash} Partition table ID
* `--puppet-ca-proxy VALUE` {endash} Name of Puppet CA proxy
* `--puppet-ca-proxy-id NUMBER` {endash} Puppet CA Capsule ID
* `--puppet-class-ids LIST` {endash} List of Puppet class ids
* `--puppet-classes LIST`
* `--puppet-environment VALUE` {endash} Name of associated Puppet environment
* `--puppet-environment-id NUMBER` {endash} ID of associated Puppet environment
* `--puppet-proxy VALUE` {endash} Name of Puppet proxy
* `--puppet-proxy-id NUMBER` {endash} Puppet Capsule ID
* `--pxe-loader ENUM` {endash} DHCP filename option (Grub2/PXELinux by default)
Possible value(s): `None`, `PXELinux BIOS`, `PXELinux UEFI`, `Grub UEFI`, `Grub2
BIOS`, `Grub2 ELF`, `Grub2 UEFI`, `Grub2 UEFI SecureBoot`, `Grub2 UEFI HTTP`,
`Grub2 UEFI HTTPS`, `Grub2 UEFI HTTPS SecureBoot`, `iPXE Embedded`, `iPXE UEFI
HTTP`, `iPXE Chain BIOS`, `iPXE Chain UEFI`
* `--query-organization VALUE` {endash} Organization name to search by
* `--query-organization-id VALUE` {endash} Organization ID to search by
* `--query-organization-label VALUE` {endash} Organization label to search by
* `--realm VALUE` {endash} Name to search by
* `--realm-id NUMBER` {endash} Realm ID
* `--root-password VALUE` {endash} Root password
* `--subnet VALUE` {endash} Subnet name
* `--subnet-id NUMBER` {endash} Subnet ID
* `--subnet6 VALUE` {endash} Subnet IPv6 name
* `--subnet6-id NUMBER` {endash} Subnet IPv6 ID
* `-h`, `--help` {endash} Print help

Following parameters accept format defined by its schema (bold are required; <> contains acceptable type; [] contains acceptable value):

* `--group-parameters-attributes`  "*name*=<string>,*value*=<string>,parameter_type=[string|boolean|integer|real|array|hash|yaml|json],hidden_value=[true|false|1|0], ... "
