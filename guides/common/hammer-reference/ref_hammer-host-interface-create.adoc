[id="hammer-host-interface-create"]
= host interface create

[role="_abstract"]
Create an interface on a host

*Usage*

----
# hammer host interface create [OPTIONS]
----

*Options*

* `--attached-devices LIST` {endash} Identifiers of attached interfaces, e.g. `[`eth1`, `eth2`]`. For bond interfaces
those are the slaves. Only for bond and bridges interfaces.
* `--attached-to VALUE` {endash} Identifier of the interface to which this interface belongs, e.g. eth1. Only for
virtual interfaces.
* `--bond-options VALUE` {endash} Space separated options, e.g. miimon=100. Only for bond interfaces.
* `--compute-attributes KEY_VALUE_LIST` Compute resource specific attributes
* `--domain VALUE` {endash} Domain name
* `--domain-id NUMBER` {endash} Satellite domain ID of interface. Required for primary interfaces on managed
hosts.
* `--execution BOOLEAN` {endash} Should this interface be used for remote execution?
* `--host VALUE` {endash} Host name
* `--host-id VALUE` {endash} ID or name of host
* `--identifier VALUE` {endash} Device identifier, e.g. eth0 or eth1.1
* `--ip VALUE` {endash} IPv4 address of interface
* `--ip6 VALUE` {endash} IPv6 address of interface
* `--location VALUE` {endash} Set the current location context for the request
* `--location-id NUMBER` {endash} Set the current location context for the request
* `--location-title VALUE` {endash} Set the current location context for the request
* `--mac VALUE` {endash} MAC address of interface. Required for managed interfaces on bare metal.
* `--managed BOOLEAN` {endash} Should this interface be managed via DHCP and DNS Capsule and should it be
configured during provisioning?
* `--mode ENUM` {endash} Bond mode of the interface, e.g. balance-rr. Only for bond interfaces.
Possible value(s): `balance-rr`, `active-backup`, `balance-xor`, `broadcast`,
`802.3ad`, `balance-tlb`, `balance-alb`
* `--mtu NUMBER` {endash} MTU, this attribute has precedence over the subnet MTU.
* `--name VALUE` {endash} Interface`s DNS name
* `--organization VALUE` {endash} Set the current organization context for the request
* `--organization-id NUMBER` {endash} Set the current organization context for the request
* `--organization-title VALUE` {endash} Set the current organization context for the request
* `--password VALUE` {endash} Only for BMC interfaces.
* `--primary` {endash} Should this interface be used for constructing the FQDN of the host? Each
managed hosts needs to have one primary interface
* `--provider ENUM` {endash} Interface provider, e.g. IPMI. Only for BMC interfaces.
Possible value(s): `IPMI`, `Redfish`, `SSH`
* `--provision` {endash} Should this interface be used for TFTP of PXELinux (or SSH for image-based
hosts)? Each managed hosts needs to have one provision interface
* `--subnet VALUE` {endash} Subnet name
* `--subnet-id NUMBER` {endash} Satellite subnet ID of IPv4 interface
* `--subnet6-id NUMBER` {endash} Satellite subnet ID of IPv6 interface
* `--tag VALUE` {endash} VLAN tag, this attribute has precedence over the subnet VLAN ID. Only for
virtual interfaces.
* `--type ENUM` {endash} Interface type, e.g. bmc. Default is interface
Possible value(s): `interface`, `bmc`, `bond`, `bridge`
* `--username VALUE` {endash} Only for BMC interfaces.
* `--virtual BOOLEAN` {endash} Alias or VLAN device
* `-h`, `--help` {endash} Print help
