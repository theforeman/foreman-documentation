[id="hammer-host-registration-generate-command"]
= host-registration generate-command

[role="_abstract"]
Generate global registration command

*Usage*

----
# hammer host-registration generate-command [OPTIONS]
----

*Options*

* `--activation-key VALUE` {endash} Activation key for subscription-manager client, required for CentOS and Red Hat
Enterprise Linux. For multiple keys use `activation_keys` param instead.
* `--activation-keys LIST` {endash} Activation keys for subscription-manager client, required for CentOS and Red Hat
Enterprise Linux. Required only if host group has no activation keys or if you
do not provide a host group.
* `--download-utility ENUM` {endash} The download utility to use during host registration
Possible value(s): `curl`, `wget`
* `--force BOOLEAN` {endash} Clear any previous registration and run subscription-manager with –force.
* `--hostgroup VALUE` {endash} Hostgroup name
* `--hostgroup-id NUMBER` {endash} ID of the Host group to register the host in
* `--hostgroup-title VALUE` {endash} Hostgroup title
* `--ignore-subman-errors BOOLEAN` {endash} Ignore subscription-manager errors for `subscription-manager register` command
* `--insecure BOOLEAN` {endash} Enable insecure argument for the initial curl/wget
* `--jwt-expiration NUMBER` {endash} Expiration of the authorization token (in hours), 0 means `unlimited`.
* `--location VALUE` {endash} Set the current location context for the request
* `--location-id NUMBER` {endash} Set the current location context for the request
* `--location-title VALUE` {endash} Set the current location context for the request
* `--operatingsystem VALUE` {endash} Operating system title
* `--operatingsystem-id NUMBER` {endash} ID of the Operating System to register the host in. Operating system must have a
`host_init_config` template assigned
* `--organization VALUE` {endash} Set the current organization context for the request
* `--organization-id NUMBER` {endash} Set the current organization context for the request
* `--organization-title VALUE` {endash} Set the current organization context for the request
* `--packages VALUE` {endash} Packages to install on the host when registered. Can be set by `host_packages`
parameter, example: `pkg1 pkg2`
* `--remote-execution-interface VALUE` {endash} Identifier of the Host interface for Remote execution
* `--repo VALUE` {endash} DEPRECATED, use the `repo_data` param instead.
* `--repo-data SCHEMA` {endash} Array with repository URL and corresponding GPG key URL
* `--repo-gpg-key-url VALUE` {endash} DEPRECATED, use the `repo_data` param instead.
* `--setup-container-registry-certs BOOLEAN` Use container certificates for container registry authentication. If it is set
to true, container registry certificates will be installed on the host
* `--setup-insights BOOLEAN` {endash} Set `host_registration_insights` parameter for the host. If it is set to true,
Red Hat Lightspeed client will be installed and registered on Red Hat family
operating systems
* `--setup-remote-execution BOOLEAN` {endash} Set `host_registration_remote_execution` parameter for the host. If it is set to
true, SSH keys will be installed on the host
* `--setup-remote-execution-pull BOOLEAN` {endash} Set `host_registration_remote_execution_pull` parameter for the host. If it is
set to true, pull provider client will be deployed on the host
* `--smart-proxy VALUE` {endash} Name to search by
* `--smart-proxy-id NUMBER` {endash} ID of the Capsule. This Capsule must have enabled both the `Templates` and
`Registration` features
* `--update-packages BOOLEAN` {endash} Update all packages on the host
* `-h`, `--help` {endash} Print help

Following parameters accept format defined by its schema (bold are required; <> contains acceptable type; [] contains acceptable value):

* `--repo-data` {endash} "repo=<string>,repo_gpg_key_url=<string>, ... "
