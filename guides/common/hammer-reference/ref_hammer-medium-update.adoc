[id="hammer-medium-update"]
= medium update

[role="_abstract"]
Update a medium

*Usage*

----
# hammer medium update [OPTIONS]
----

*Options*

* `--id VALUE`
* `--location VALUE` {endash} Set the current location context for the request
* `--location-id NUMBER` {endash} Set the current location context for the request
* `--location-ids LIST` {endash} REPLACE locations with given ids
* `--location-title VALUE` {endash} Set the current location context for the request
* `--location-titles LIST`
* `--locations LIST`
* `--name VALUE` {endash} Name of media
* `--new-name VALUE` {endash} Name of media
* `--operatingsystem-ids LIST`
* `--operatingsystems LIST`
* `--organization VALUE` {endash} Set the current organization context for the request
* `--organization-id NUMBER` {endash} Set the current organization context for the request
* `--organization-ids LIST` {endash} REPLACE organizations with given ids.
* `--organization-title VALUE` {endash} Set the current organization context for the request
* `--organization-titles LIST`
* `--organizations LIST`
* `--os-family VALUE` {endash} Operating system family, available values: AIX, Altlinux, Archlinux, Coreos,
Debian, Fcos, Freebsd, Gentoo, Junos, NXOS, Rancheros, Redhat, Rhcos, Solaris,
Suse, VRP, Windows, Xenserver
* `--path VALUE` {endash} The path to the medium, can be a URL or a valid NFS server (exclusive of the
architecture).

For example mirror.centos.org/centos/$version/os/$arch where $arch will be
substituted for the host`s actual OS architecture and $version, $major and
$minor will be substituted for the version of the operating system.

Solaris and Debian media may also use $release.
* `-h`, `--help` {endash} Print help
