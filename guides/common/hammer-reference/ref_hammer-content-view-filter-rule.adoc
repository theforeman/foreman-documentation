= content-view filter rule

[role="_abstract"]
View and manage filter rules

*Usage*

----
# hammer content-view filter rule [OPTIONS] SUBCOMMAND [ARG] ...
----



*Options*

* `-h`, `--help` {endash} Print help



[id="hammer-content-view-filter-rule-create"]
== content-view filter rule create

Create a filter rule. The parameters included should be based upon the filter type.

*Usage*

----
# hammer content-view filter rule create [OPTIONS]
----

*Options*

* `--allow-other-types BOOLEAN` {endash} Erratum: allow types not matching a valid errata type
* `--architecture VALUE` {endash} Package: architecture
* `--content-view VALUE` {endash} Content view name to search by
* `--content-view-filter VALUE` {endash} Name to search by
* `--content-view-filter-id NUMBER` Filter identifier
* `--content-view-id NUMBER`
* `--date-type VALUE` {endash} Erratum: search using the `Issued On` or `Updated On` column of the errata.
Values are `issued`/`updated`
* `--end-date VALUE` {endash} Erratum: end date (YYYY-MM-DD)
* `--errata-id VALUE` {endash} Erratum: id
* `--errata-ids LIST` {endash} Erratum: IDs or a select all object
* `--max-version VALUE` {endash} Package: maximum version
* `--min-version VALUE` {endash} Package: minimum version
* `--module-stream-ids LIST` {endash} Module stream ids
* `--name LIST` {endash} Deb, package, package group, or docker tag names
* `--names VALUE` {endash} Package and package group names
* `--organization VALUE` {endash} Organization name to search by
* `--organization-id VALUE` {endash} Organization ID to search by
* `--organization-label VALUE` {endash} Organization label to search by
* `--organization-title VALUE` {endash} Organization title to search by
* `--start-date VALUE` {endash} Erratum: start date (YYYY-MM-DD)
* `--types LIST` {endash} Erratum: types (enhancement, bugfix, security)
* `--uuid VALUE` {endash} Package group: uuid
* `--version VALUE` {endash} Package: version
* `-h`, `--help` {endash} Print help
