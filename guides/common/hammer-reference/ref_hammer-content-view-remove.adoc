[id="hammer-content-view-remove"]
= content-view remove

[role="_abstract"]
Remove versions and/or environments from a content view and reassign systems and keys

*Usage*

----
# hammer content-view remove [OPTIONS]
----

*Options*

* `--async` {endash} Do not wait for the task
* `--content-view-version-ids LIST`  Content view version identifiers to be deleted
* `--content-view-versions LIST`
* `--destroy-content-view BOOLEAN` {endash} Delete the content view with all the versions and environments
* `--environment-ids LIST` {endash} (--environment-ids is deprecated: Use `--lifecycle-environment-ids` instead)
* `--environments LIST` {endash} (--environments is deprecated: Use `--lifecycle-environments` instead)
* `--id NUMBER` {endash} Content view numeric identifier
* `--key-content-view-id NUMBER` {endash} Content view to reassign orphaned activation keys to
* `--key-environment-id NUMBER` {endash} Environment to reassign orphaned activation keys to
* `--lifecycle-environment-ids LIST` Environment numeric identifiers to be removed
* `--lifecycle-environments LIST`
* `--name VALUE` {endash} Content view name to search by
* `--organization VALUE` {endash} Organization name to search by
* `--organization-id VALUE` {endash} Organization ID to search by
* `--organization-label VALUE` {endash} Organization label to search by
* `--organization-title VALUE` {endash} Organization title to search by
* `--system-content-view-id NUMBER`  Content view to reassign orphaned systems to
* `--system-environment-id NUMBER` {endash} Environment to reassign orphaned systems to
* `-h`, `--help` {endash} Print help
