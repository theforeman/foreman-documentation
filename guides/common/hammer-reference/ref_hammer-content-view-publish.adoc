[id="hammer-content-view-publish"]
= content-view publish

[role="_abstract"]
Publish a content view

*Usage*

----
# hammer content-view publish [OPTIONS]
----

*Options*

* `--async` {endash} Do not wait for the task
* `--description VALUE` {endash} Description for the new published content view version
* `--id NUMBER` {endash} Content view identifier
* `--is-force-promote BOOLEAN` {endash} Force content view promotion and bypass lifecycle environment restriction
* `--lifecycle-environment-ids LIST` Identifiers for Lifecycle Environment
* `--lifecycle-environments LIST`
* `--major NUMBER` {endash} Override the major version number
* `--minor NUMBER` {endash} Override the minor version number
* `--name VALUE` {endash} Content view name to search by
* `--organization VALUE` {endash} Organization name to search by
* `--organization-id VALUE` {endash} Organization ID to search by
* `--organization-label VALUE` {endash} Organization label to search by
* `--organization-title VALUE` {endash} Organization title to search by
* `--publish-only-if-needed BOOLEAN` Check audited changes and proceed only if content or filters have changed since
last publish
* `--repos-units SCHEMA` {endash} Specify the list of units in each repo
* `-h`, `--help` {endash} Print help

Following parameters accept format defined by its schema (bold are required; <> contains acceptable type; [] contains acceptable value):

* `--repos-units` {endash} "*label*=<string>,*rpm_filenames*=<array>, ... "
