= content-view filter create

[role="_abstract"]
create a filter for a content view

*Usage*

----
# hammer content-view filter create [OPTIONS]
----

*Options*

* `--content-view VALUE` {endash} Content view name to search by
* `--content-view-id NUMBER` {endash} Content view identifier
* `--description VALUE` {endash} Description of the filter
* `--inclusion BOOLEAN` {endash} Specifies if content should be included or excluded, default: inclusion=false
* `--name VALUE` {endash} Name of the filter
* `--organization VALUE` {endash} Organization name to search by
* `--organization-id VALUE` {endash} Organization ID to search by
* `--organization-label VALUE` {endash} Organization label to search by
* `--organization-title VALUE` {endash} Organization title to search by
* `--original-module-streams BOOLEAN` Add all module streams without errata to the included/excluded list. (module
stream filter only)
* `--original-packages BOOLEAN` {endash} Add all packages without errata to the included/excluded list. (package filter
only)
* `--product VALUE` {endash} Product name to search by
* `--product-id NUMBER` {endash} Product numeric identifier
* `--repositories LIST`
* `--repository-ids LIST` {endash} List of repository ids
* `--type VALUE` {endash} Type of filter (e.g. deb, rpm, package_group, erratum, erratum_id, erratum_date,
docker, modulemd)
* `-h`, `--help` {endash} Print help
