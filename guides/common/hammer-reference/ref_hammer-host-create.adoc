= host create

[role="_abstract"]
Create a host

*Usage*

----
# hammer host create [OPTIONS]
----

*Options*

* `--ansible-role-ids LIST` {endash} IDs of associated ansible roles
* `--ansible-roles LIST`
* `--architecture VALUE` {endash} Architecture name
* `--architecture-id NUMBER` {endash} Required if host is managed and value is not inherited from host group
* `--ask-root-password BOOLEAN`
* `--autoheal BOOLEAN` {endash} Sets whether the Host will autoheal subscriptions upon checkin
* `--build BOOLEAN`
* `--comment VALUE` {endash} Additional information about this host
* `--compute-attributes KEY_VALUE_LIST` {endash} Compute resource attributes
* `--compute-profile VALUE` {endash} Compute profile name
* `--compute-profile-id NUMBER`
* `--compute-resource VALUE` {endash} Compute resource name
* `--compute-resource-id NUMBER` {endash} Nil means host is bare metal
* `--config-group-ids LIST` {endash} IDs of associated config groups
* `--config-groups LIST`
* `--content-source VALUE` {endash} Content Source name
* `--content-source-id NUMBER` {endash} Id of the Capsule from which the host consumes content.
* `--content-view VALUE` {endash} Name to search by
* `--content-view-environment-ids LIST` {endash} Array of content view environment ids to be associated with the host. Ignored if
content_view_id and lifecycle_environment_id are specified. Requires
allow_multiple_content_views setting to be on.
* `--content-view-environments LIST` {endash} Comma-separated list of content view environment labels to be associated with
the host, in the format of `lifecycle_environment_label/content_view_label`.
Ignored if content_view_environment_ids is specified, or if content_view_id and
lifecycle_environment_id are specified. Requires allow_multiple_content_views
setting to be on.
* `--content-view-id NUMBER` {endash} Id of the single content view to be associated with the host.
* `--domain VALUE` {endash} Domain name
* `--domain-id NUMBER` {endash} Required if host is managed and value is not inherited from host group
* `--enabled BOOLEAN` {endash} Include this host within Satellite reporting
* `--hostgroup VALUE` {endash} Hostgroup name
* `--hostgroup-id NUMBER`
* `--hostgroup-title VALUE` {endash} Hostgroup title
* `--hypervisor-guest-uuids LIST` {endash} List of hypervisor guest uuids
* `--image VALUE` {endash} Name to search by
* `--image-id NUMBER`
* `--installed-products-attributes SCHEMA` List of products installed on the host
* `--interface KEY_VALUE_LIST` {endash} Interface parameters
Can be specified multiple times.
* `--ip VALUE` {endash} Not required if using a subnet with DHCP Capsule
* `--kickstart-repository VALUE` {endash} Kickstart repository name
* `--kickstart-repository-id NUMBER` {endash} Repository Id associated with the kickstart repo used for provisioning
* `--lifecycle-environment VALUE` {endash} Name to search by
* `--lifecycle-environment-id NUMBER` {endash} Id of the single lifecycle environment to be associated with the host.
* `--location VALUE` {endash} Set the current location context for the request
* `--location-id NUMBER` {endash} Set the current location context for the request
* `--location-title VALUE` {endash} Set the current location context for the request
* `--mac VALUE` {endash} Required for managed host that is bare metal, not required if it`s a virtual
machine
* `--managed BOOLEAN` {endash} True/False flag whether a host is managed or unmanaged. Note: this value also
determines whether several parameters are required or not
* `--medium VALUE` {endash} Medium name
* `--medium-id VALUE` {endash} Required if not imaged based provisioning and host is managed and value is not
inherited from host group
* `--model VALUE` {endash} Model name
* `--model-id NUMBER`
* `--name VALUE`
* `--openscap-proxy-id NUMBER` {endash} ID of OpenSCAP Capsule
* `--operatingsystem VALUE` {endash} Operating system title
* `--operatingsystem-id NUMBER` {endash} Required if host is managed and value is not inherited from host group
* `--organization VALUE` {endash} Set the current organization context for the request
* `--organization-id NUMBER` {endash} Set the current organization context for the request
* `--organization-title VALUE` {endash} Set the current organization context for the request
* `--overwrite BOOLEAN` {endash} Default: "true"
* `--owner VALUE` {endash} Login of the owner
* `--owner-id VALUE` {endash} ID of the owner
* `--owner-type ENUM` {endash} Host`s owner type
Possible value(s): `User`, `Usergroup`
* `--parameters KEY_VALUE_LIST` {endash} Replaces with new host parameters
* `--partition-table VALUE` {endash} Partition table name
* `--partition-table-id NUMBER` {endash} Required if host is managed and custom partition has not been defined
* `--product VALUE` {endash} Name to search by
* `--product-id NUMBER` {endash} Product id as listed from a host`s installed products,         this is not the
same product id as the products api returns
* `--progress-report-id VALUE` {endash} UUID to track orchestration tasks status, GET /api/orchestration/:UUID/tasks
* `--provision-method ENUM` {endash} The method used to provision the host.
Possible value(s): `build`, `image`, `bootdisk`
* `--puppet-ca-proxy VALUE` {endash} Name of Puppet CA proxy
* `--puppet-ca-proxy-id NUMBER` {endash} Puppet CA Capsule ID
* `--puppet-class-ids LIST` {endash} List of Puppet class ids
* `--puppet-classes LIST`
* `--puppet-environment VALUE` {endash} Name of associated Puppet environment
* `--puppet-environment-id NUMBER` {endash} ID of associated Puppet environment
* `--puppet-proxy VALUE` {endash} Name of Puppet proxy
* `--puppet-proxy-id NUMBER` {endash} Puppet Capsule ID
* `--purpose-role VALUE` {endash} Sets the system purpose usage
* `--purpose-usage VALUE` {endash} Sets the system purpose usage
* `--pxe-loader ENUM` {endash} DHCP filename option (Grub2/PXELinux by default)
Possible value(s): `None`, `PXELinux BIOS`, `PXELinux UEFI`, `Grub UEFI`, `Grub2
BIOS`, `Grub2 ELF`, `Grub2 UEFI`, `Grub2 UEFI SecureBoot`, `Grub2 UEFI HTTP`,
`Grub2 UEFI HTTPS`, `Grub2 UEFI HTTPS SecureBoot`, `iPXE Embedded`, `iPXE UEFI
HTTP`, `iPXE Chain BIOS`, `iPXE Chain UEFI`
* `--realm VALUE` {endash} Name to search by
* `--realm-id NUMBER`
* `--release-version VALUE` {endash} Release version for this Host to use (7Server, 7.1, etc)
* `--root-password VALUE` {endash} Required if host is managed and value is not inherited from host group or
default password in settings
* `--service-level VALUE` {endash} Service level to be used for autoheal
* `--subnet VALUE` {endash} Subnet name
* `--subnet-id NUMBER` {endash} Required if host is managed and value is not inherited from host group
* `--typed-parameters SCHEMA` {endash} Replaces with new host parameters (with type support)
* `--volume KEY_VALUE_LIST` {endash} Volume parameters
Can be specified multiple times.
* `-h`, `--help` {endash} Print help

Following parameters accept format defined by its schema (bold are required; <> contains acceptable type; [] contains acceptable value):

* `--installed-products-attributes`  "product_id=<string>,product_name=<string>,arch=<string>,version=<string>, ... "
