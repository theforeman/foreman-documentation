= Configuring service log levels at runtime

[role="_abstract"]
You can temporarily change the log level of a running {Project} service without redeploying, which is useful for short-lived debugging sessions.
If an error occurs in production, you can temporarily elevate the log level to `debug` to capture detailed failure details, resolve the issue, and then drop it back down to `info` to avoid filling up host disk space.


.Prerequisites
* {ProjectServer} is installed and running.
* You have root access to the {ProjectServer} host.

.Procedure
* To change the runtime log level of running containers, run:
+
[options="nowrap", subs="+quotes,verbatim,attributes"]
----
# {foremanctl} logs set-level debug
----

.Verification
