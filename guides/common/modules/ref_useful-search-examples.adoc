= Useful search examples

[role="_abstract"]
These examples demonstrate powerful ways to filter and find resources in {ProjectWebUI}.
Use them as starting points to build your own searches that match your specific needs.

*Find hosts that need attention*

Hosts with security updates available:

[options="nowrap", subs="+quotes,attributes"]
----
errata_status = security_needed
----

Hosts that are enabled but in a failed state:

[options="nowrap", subs="+quotes,attributes"]
----
status.enabled = true AND status.failed
----

Hosts that haven't checked in recently:

[options="nowrap", subs="+quotes,attributes"]
----
last_report < "7 days ago"
----

*Find hosts by configuration or environment*

Hosts in production that belong to a specific host group:

[options="nowrap", subs="+quotes,attributes"]
----
environment = production AND hostgroup ~ webservers
----

Hosts running a specific OS version:

[options="nowrap", subs="+quotes,attributes"]
----
os_title ~ "RedHat 9"
----

Hosts with a specific fact value (for example, finding all virtual machines):

[options="nowrap", subs="+quotes,attributes"]
----
facts.virtual = true
----

*Find hosts by resource usage or quota*

Hosts without an assigned resource quota:

[options="nowrap", subs="+quotes,attributes"]
----
resource_quota = Unassigned
----

Hosts consuming more than a certain number of CPUs:

[options="nowrap", subs="+quotes,attributes"]
----
facts.processors.count > 8
----

*Find hosts by time and registration*

Hosts registered in the last month:

[options="nowrap", subs="+quotes,attributes"]
----
registered_at > "30 days ago"
----

Hosts registered between specific dates:

[options="nowrap", subs="+quotes,attributes"]
----
registered_at > 1-January-2026 AND registered_at < 31-January-2026
----

*Combine multiple criteria*

Production web servers that need security updates:

[options="nowrap", subs="+quotes,attributes"]
----
environment = production AND hostgroup ~ web AND errata_status = security_needed
----

Virtual machines in a specific location without assigned host group:

[options="nowrap", subs="+quotes,attributes"]
----
facts.virtual = true AND location = "Data Center 1" AND not has hostgroup
----

Hosts pending installation or with installation errors:

[options="nowrap", subs="+quotes,attributes"]
----
build_status = "Pending installation" OR build_status = "Installation error"
----

*Exclude specific results*

All hosts except those in a specific host group:

[options="nowrap", subs="+quotes,attributes"]
----
NOT hostgroup = exceptions
----

Hosts that are not compliant with policies:

[options="nowrap", subs="+quotes,attributes"]
----
compliance_status != "Compliant"
----

*Use wildcards for flexible matching*

Hosts with names matching a pattern:

[options="nowrap", subs="+quotes,attributes"]
----
name ~ web*prod*
----

Host groups starting with a specific prefix:

[options="nowrap", subs="+quotes,attributes"]
----
hostgroup ~ rhel%
----

[TIP]
====
Click the search field on any page to see all available search parameters for that specific context.
The autocomplete feature suggests valid field names, operators, and values as you type.
====
