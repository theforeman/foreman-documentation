= Customize values of Ansible variables using simple matchers by using {ProjectWebUI}

[role="_abstract"]
You can set custom values of Ansible variables in {Project} by using {ProjectWebUI}.
Simple matchers help you override values for specific hosts by FQDN or by host group.

.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_variable_overrides`, `destroy_ansible_variable_overrides`, `destroy_ansible_variables`, `edit_ansible_variable_overrides`, `edit_ansible_variables`, `view_ansible_content`, `view_ansible_variable_overrides`, `view_ansible_variables`, and `view_organizations` permissions.
* Your {Project} contains imported Ansible content.

.Procedure
. In the {ProjectWebUI}, navigate to *Ansible* > *Content*.
. From the list of Ansible content, select a version.
. Select an Ansible role.
. Use the toggle to enable the variable override.
. Click the *Edit* icon.
. On the *Overrides* tab, click *Create override*.
. Set the toggle to *Simple Matcher Creation*.
. In the *Matcher type* menu, select your simple matcher type.
. In the *Matcher value* menu, select your simple matcher value.
. In the *Override value* field, enter your custom value.
. Click *Submit* to save your override.

.Verification
. In the {ProjectWebUI}, navigate to *Ansible* > *Content*.
. From the list of Ansible content, select a version.
. Select an Ansible role.
. Use the toggle to enable the variable override.
. Click the *Edit* icon.
. Verify that {Project} contains your custom value.
