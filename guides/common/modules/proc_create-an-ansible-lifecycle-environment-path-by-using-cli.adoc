= Create an Ansible lifecycle environment path by using Hammer CLI

[role="_abstract"]
You can create an Ansible lifecycle environment path by using Hammer CLI.
Similar to content management, Ansible lifecycle environment paths allow you to develop, test, and run the same Ansible content in production in different versions.

.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_lifecycle_environment_paths`, `edit_ansible_lifecycle_environment_paths`, `view_ansible_content`, `view_ansible_lifecycle_environment_paths`, `view_ansible_lifecycle_environments`, and `view_organizations` permissions.
* You have installed the Hammer CLI plugin for Ansible Director.
For more information, see xref:install-the-hammer-cli-plugin-for-ansible-director[].

.Procedure
* Create an Ansible lifecycle environment path:
+
----
$ hammer
----

.Verification
* Verify that {Project} created the Ansible lifecycle environment path successfully:
+
----
$ hammer
----
