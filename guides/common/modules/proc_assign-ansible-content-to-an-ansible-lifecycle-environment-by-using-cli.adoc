= Assign Ansible content to an Ansible lifecycle environment by using Hammer CLI

[role="_abstract"]
You can assign Ansible content to an Ansible lifecycle environment by using Hammer CLI.
Ansible content consists of imported Ansible collections and roles in specific versions.

.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_lifecycle_environment_paths`, `edit_ansible_lifecycle_environment_paths`, `view_ansible_content`, `view_ansible_lifecycle_environment_paths`, `view_ansible_lifecycle_environments`, and `view_organizations` permissions.
* Your {Project} contains an Ansible lifecycle environment path.
* Your {Project} contains Ansible content.
* You have installed the Hammer CLI plugin for Ansible Director.
For more information, see xref:install-the-hammer-cli-plugin-for-ansible-director[].

.Procedure
. Optional: List all Ansible lifecycle environments with your Ansible lifecycle environment path in {Project}:
+
----
$ hammer
----
. Assign Ansible content to your Ansible lifecycle environment:
+
----
$ hammer
----

.Verification
* Verify that {Project} assigned the Ansible content to your Ansible lifecycle environment successfully:
+
----
$ hammer
----
