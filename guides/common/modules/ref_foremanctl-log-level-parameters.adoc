[id="foremanctl-deploy-log-level-parameters"]
= `{foremanctl} deploy` log level parameters

[role="_abstract"]
`{foremanctl} deploy` accepts log level parameters that control the logging verbosity of containerized {Project} services.
You can set a global baseline and override it for individual services.

.Log level parameters
[cols="2,2,1,2,2", options="header"]
|===
| Parameter
| Valid values
| Default
| Affected service
| Cascade behavior

| `--log-level`
| `debug`, `info`, `warn`, `error`, `fatal`
| `info`
| All services that support per-service log levels
| Sets the global baseline.
Service-specific parameters override this value.

| `--foreman-log-level`
| `debug`, `info`, `warn`, `error`, `fatal`
| Value of `--log-level`
| `foreman`
| Overrides `--log-level` for the Foreman service only.

| `--foreman-proxy-log-level`
| `debug`, `info`, `warn`, `error`, `fatal`
| Value of `--log-level`
| `foreman-proxy`
| Overrides `--log-level` for {SmartProxy} only.

ifdef::satellite,katello,orcharhino[]
| `--valkey-log-level`
| `debug`, `verbose`, `notice`, `warning`, `nothing`
| `notice`
| `valkey`
| Independent of `--log-level`.
Uses the Valkey-specific vocabulary.
endif::[]
|===

ifdef::satellite,katello,orcharhino[]
== Valkey log level mapping

.Standard to Valkey log level mapping
[cols="1,1", options="header"]
|===
| Standard level | Valkey equivalent

| `debug`
| `debug`

| `info`
| `notice`

| `warn`
| `warning`

| `error`
| `warning`

| `fatal`
| `warning`
|===

The `verbose` and `nothing` levels are available only through the `--valkey-log-level` parameter and have no standard equivalent.
endif::[]

== Traditional parameter equivalents

.Traditional to containerized parameter mapping
[cols="1,1", options="header"]
|===
| Traditional parameter | Containerized equivalent

| `--foreman-logging-level`
| `--foreman-log-level`

| `--foreman-proxy-log`
| `--foreman-proxy-log-level`

| `--foreman-logging-layout`
| No equivalent

| `--foreman-logging-type`
| No equivalent
|===
