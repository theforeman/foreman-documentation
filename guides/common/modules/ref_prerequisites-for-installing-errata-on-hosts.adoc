= Prerequisites for installing errata on hosts

[role="_abstract"]
Installing errata on hosts requires synchronized repositories, installable errata in the content view environment, registered hosts, and remote execution configuration.

ifdef::satellite[]
* Synchronize repositories on {ProjectServer} with the latest errata available from Red{nbsp}Hat.
endif::[]
ifndef::satellite[]
* Synchronize repositories on {ProjectServer} with the latest errata available from upstream.
endif::[]
* Ensure errata are *installable*.
Applicable errata have updates for packages on hosts, but you can only install errata included in the content view environment of the host.
If errata are applicable but not installable, create an incremental content view update and promote the content view to the lifecycle environment of the host.
* Register the host to a content view environment.
* Configure the host for remote execution.
