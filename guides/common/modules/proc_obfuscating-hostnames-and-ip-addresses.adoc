= Obfuscating hostnames and IP addresses

[role="_abstract"]
You can configure {Project} to obfuscate hostnames and IP addresses when uploading host data to {Insights}. 
Obfuscation ensures that sensitive identifying information is not exposed. 

When obfuscation settings are enabled, {Project} obfuscates hostnames and IP addresses. 
If obfuscation settings are enabled and the host has its own `insights-client` obfuscation settings, {Project} honors the host settings.  
If obfuscation settings are disabled, {Project} does not obfuscate hostnames or IP addresses unless the client has obfuscation enabled in `insights-client`.

By default, {Project} applies global obfuscation settings unless an `insights-client` on the host overrides the settings with its own configuration.

[NOTE]
====
Obfuscation only works when *Minimal data collection* is disabled.
If *Minimal data collection* is enabled, hostnames and IP addresses are not included in reports, making obfuscation unnecessary and inactive.
====

.Prerequisites
* Your hosts have `insights-client` installed and configured.

.Procedure
. In the {ProjectWebUI}, navigate to *{Insights}* > *Inventory Upload*.
. Enable the *Obfuscate host names* setting to obfuscate host names sent to the {RHCloud}.
. Enable the *Obfuscate host ipv4 addresses* setting to obfuscate IPv4 addresses sent to the {RHCloud}.
. For the new settings to take effect, generate and upload a new host inventory report:
.. Click *Generate and upload report* to create and upload a new report with obfuscated data.
.. Wait for the report generation and upload to complete.

.Verification
* In the {RHCloud}, verify that the hostnames and IP addresses are obfuscated.
