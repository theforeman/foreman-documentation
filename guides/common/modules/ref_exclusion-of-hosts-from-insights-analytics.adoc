= Exclusion of hosts from {Insights} analytics

[role="_abstract"]
You can exclude hosts from {Insights} analytics by using
ifdef::insights-in-cloud[]
host parameters.
endif::[]
ifndef::insights-in-cloud[]
a host parameter.
endif::[]

`host_registration_insights`::
Setting this parameter to `false` will prevent {ProjectServer} from forwarding data from the Insights client on hosts to {Insights} services.
The Insights client will not be installed on newly registered hosts.

ifdef::insights-in-cloud[]
`host_registration_insights_inventory`::
Setting this parameter to `false` will exclude the host from the {Project} host inventory that is uploaded to the {RHCloud}.
+
If you exclude a host that is already reported on the https://console.redhat.com/[{RHCloud}], it will be still removed automatically from the inventory.
However, this process can take some time to complete.
endif::[]

You can set
ifdef::insights-in-cloud[]
these parameters
endif::[]
ifndef::insights-in-cloud[]
this parameter
endif::[]
at any level.
For more information, see xref:Host_Parameter_Hierarchy_managing-hosts[].
