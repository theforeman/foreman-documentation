= Adding {customrpm} repositories by using Hammer CLI

[role="_abstract"]
Create {customrpm} repositories with Hammer CLI so {Project} can synchronize Yum content into your product without opening the Web UI.

ifdef::satellite[]
[NOTE]
====
Red{nbsp}Hat does not support the upstream RPMs directly from third-party sites.
These RPMs are used to demonstrate the synchronization process.
For any issues with these RPMs, contact the third-party developers.
====
endif::[]

.Prerequisites
* You have imported the GPG key that you want to use to verify signatures of the RPM packages.
For more information, see xref:importing-a-custom-gpg-key-by-using-cli[].

.Procedure
* Create a Yum repository:
+
[options="nowrap" subs="+quotes"]
----
$ hammer repository create \
--arch "_My_Architecture_" \
--content-type "yum" \
--gpg-key-id _My_GPG_Key_ID_ \
--name "_My_Repository_" \
--organization "_My_Organization_" \
--os-version "_My_Operating_System_Version_" \
--product "_My_Product_" \
--publish-via-http true \
--url _My_Upstream_URL_
----
