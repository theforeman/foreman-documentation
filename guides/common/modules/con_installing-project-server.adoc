[id="installing-{project-context}-server"]
= Installing {ProjectServer}

[role="_abstract"]
ifdef::satellite[]
ifdef::installing-satellite-server-connected[]
Use the following procedures to install {ProjectServer} in a connected environment, perform the initial configuration, and import subscription manifests.
For more information on subscription manifests, see {ContentManagementDocURL}Managing_Red_Hat_Subscriptions_content-management[Managing Red Hat Subscriptions] in _{ContentManagementDocTitle}_.

When you install {ProjectServer} from a connected network, you can obtain packages and receive updates directly from the Red{nbsp}Hat Content Delivery Network.
endif::[]

ifdef::installing-satellite-server-disconnected[]
Use the following procedures to install {ProjectServer} in a disconnected environment, perform the initial configuration, and import subscription manifests.

When the intended host for {ProjectServer} is in a disconnected environment, you can install {ProjectServer} by using an external computer to download an ISO image of the packages, and copying the packages to the system you want to install {ProjectServer} on.
This method is not recommended for any other situation as ISO images might not contain the latest updates, bug fixes, and functionality.

ifndef::foreman-deb[]
Before you continue, consider which manifests are relevant for your environment.
For more information on manifests, see {ContentManagementDocURL}Managing_Red_Hat_Subscriptions_content-management[Managing Red Hat Subscriptions] in _{ContentManagementDocTitle}_.
endif::[]
endif::[]

ifndef::satellite[]
Use the following procedures to install {ProjectServer} and perform the initial configuration.
endif::[]

[NOTE]
====
You cannot register {ProjectServer} to itself.
====

ifdef::foreman-el[]
On {EL}, you can install {Project} with or without the Katello plugin.
If you are a new user, consider installing {Project} with the Katello plugin.
endif::[]
