= Triggering remote execution jobs in batches

[role="_abstract"]
You can control the number of tasks your {SmartProxy} recieves at once from starting a remote execution job.
This helps you to prevent overloading your {SmartProxies} when starting remote execution jobs that target many hosts.
By default, the batch size is set to `100`.

.Procedure
. In the {ProjectWebUI}, navigate to *Administer* > *Settings*.
ifdef::satellite[]
. On the *Tasks* tab, set *Allow {SmartProxy} batch tasks* to `Yes`.
endif::[]
ifndef::satellite[]
. On the *Tasks* tab, set *Allow proxy batch tasks* to `Yes`.
endif::[]
