= Migrating from hosted to in-{Project} {Insights}

[role="_abstract"]
You can migrate from hosted {Insights} to {insights-iop}.

.Prerequisites
* Integration with hosted {Insights} is enabled on your {ProjectServer}.
ifdef::satellite[]
This is enabled by default after installing {Project}.
endif::[]
* The container images of {insights-iop} are available to your {ProjectServer}.
For more information, see the following resources:
** {InstallingServerDocURL}installing-and-configuring-{insights-iop-id}[Installing and configuring {insights-iop}] in _{InstallingServerDocTitle}_
** {InstallingServerDisconnectedDocURL}installing-and-configuring-{insights-iop-id}[Installing and configuring {insights-iop}] in _{InstallingServerDisconnectedDocTitle}_

.Procedure
. On {ProjectServer}, enable {insights-iop}:
** If {insights-iop} was previously enabled:
+
[options="nowrap", subs="+quotes,verbatim,attributes"]
----
# {foreman-installer} --iop-ensure present
----
** If {insights-iop} was not previously enabled:
+
[options="nowrap", subs="+quotes,verbatim,attributes"]
----
# {foreman-installer} --enable-iop
----
. Re-register the Insights client on your hosts:
+
[options="nowrap", subs="+quotes,verbatim,attributes"]
----
# insights-client --register
----
+
You can use remote execution to run this command on your hosts.

.Verification
1. Verify that the {Insights} submenu in the {ProjectWebUI} no longer shows the *Inventory Upload* menu item and shows the *Vulnerability* menu item.
2. Verify that the same recommendations appear for your hosts as before.

.Next steps
* Run `insights-client` on your hosts to start reporting to {Project}.
