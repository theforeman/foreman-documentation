= Create an Ansible lifecycle environment path by using {Project} API

[role="_abstract"]
You can create an Ansible lifecycle environment path by using {Project} API.
Similar to content management, Ansible lifecycle environment paths allow you to develop, test, and run the same Ansible content in production in different versions.

.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_lifecycle_environment_paths`, `edit_ansible_lifecycle_environment_paths`, `view_ansible_content`, `view_ansible_lifecycle_environment_paths`, `view_ansible_lifecycle_environments`, and `view_organizations` permissions.

.Procedure
. Create a JSON file, for example, `data.json` to define your Ansible lifecycle environment path:
+
[source, json, options="nowrap", subs="verbatim,quotes,attributes"]
----
{
  "lifecycle_environment_path": {
    "name": "_My_Ansible_Lifecycle_Environment_Path_Name_"
  },
  "organization_id": "_My_Organization_ID_"
}
----
. Create an Ansible lifecycle environment path:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--data @data.json \
--header "Accept: application/json" \
--header "Content-Type: application/json" \
--request POST \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/ansible_director/lifecycle_environments/paths/ \
| jq
----

.Verification
* Query the {Project} API to verify that {Project} created the Ansible lifecycle environment path successfully:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--header "Accept: application/json" \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/ansible_director/lifecycle_environments/paths/ \
| jq ".results | .[] | .name"
----
