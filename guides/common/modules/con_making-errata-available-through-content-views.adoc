= Making errata available through content views

[role="_abstract"]
Inspect errata and make them available to hosts through content views to keep hosts up to date and secure.
Errata are software update advisories that describe patches for security issues, bug fixes, or feature enhancements.

There are three types of advisories, listed in order of importance:

Security Advisory::
Describes fixed security issues found in the package.
The severity of the issue can be Low, Moderate, Important, or Critical.

Bug Fix Advisory::
Describes bug fixes for the package.

Product Enhancement Advisory::
Describes enhancements and new features added to the package.

Errata are labeled according to the most important advisory type they contain.
Therefore, errata labeled as *Product Enhancement Advisory* can contain only enhancement updates, while *Bug Fix Advisory* errata can contain both bug fixes and enhancements, and *Security Advisory* can contain all three types.

ifdef::katello[]
{RHEL} users receive errata for each release of official Red Hat RPMs.
{SLES} users receive errata for each release of official SUSE RPMs.
endif::[]
ifdef::satellite[]
As a part of quality control and release process, Red{nbsp}Hat provides customers with updates for each release of official Red Hat RPMs.
Red{nbsp}Hat compiles groups of related packages into an _erratum_ along with an advisory that provides a description of the update.
endif::[]
ifdef::orcharhino[]
Depending on the operating systems your hosts run, you receive errata from the following sources:

* AlmaLinux users receive errata from the upstream project.
* Amazon Linux users receive errata from Amazon.
* CentOS 7 users receive errata from ATIX.
* CentOS Stream users receive errata from the upstream project.
* Debian users receive errata from ATIX.
* Oracle Linux users receive errata for each release of official Oracle RPMs.
* Red Hat Enterprise Linux users receive errata for each release of official Red Hat RPMs.
* Rocky Linux users receive errata from the upstream project.
* SUSE Linux Enterprise Server users receive errata for each release of official SUSE RPMs.
* Ubuntu users receive errata from ATIX.
endif::[]

ifdef::katello[]
For {RHEL}, {Project} imports errata information when synchronizing repositories from the Red{nbsp}Hat Content Delivery Network (CDN).
For {SLES}, {Project} imports errata information when synchronizing repositories from the SUSE Customer Center.
For AlmaLinux and Rocky Linux, {Project} imports errata information when synchronizing repositories from the upstream repositories.
endif::[]
ifdef::satellite[]
{ProjectName} imports this errata information when synchronizing repositories from the Red{nbsp}Hat Content Delivery Network (CDN).
endif::[]
ifdef::orcharhino[]
{ProjectName} imports this errata information when synchronizing repositories from upstream.
endif::[]

In {ProjectName}, there are two keywords that describe the relationship of an erratum to hosts:

Applicable::
An erratum that applies to one or more hosts, which means it has updates for packages present on the host.
Although these errata apply to hosts, until their state changes to *Installable*, the errata are not ready to be installed.
Publish incremental content views to make applicable errata installable.
Installable errata are always applicable.

Installable::
An erratum that applies to one or more hosts and is ready to be installed on the host.
Installable errata are available to a host from its associated content view environments, but are not yet installed.
