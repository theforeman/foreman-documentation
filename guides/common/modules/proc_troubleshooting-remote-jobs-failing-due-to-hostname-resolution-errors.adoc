= Troubleshooting remote jobs failing due to hostname resolution errors

[role="_abstract"]
If remote execution jobs fail on your host due to `Could not resolve hostname: Name or service not known`, disable *Ignore interfaces facts for provisioning*.

.Procedure
. In the {ProjectWebUI}, navigate to *Hosts* > *All Hosts*.
. Select your host.
. Click *Edit*.
. On the *Interfaces* tab, check if the host has an IP address.
If the host has no IP address, continue with this procedure.
. Navigate to *Administer* > *Settings*.
. On the *Facts* tab, set *Ignore interfaces facts for provisioning* to `No`.
This ensures that an IP address is assigned to the host during registration.
