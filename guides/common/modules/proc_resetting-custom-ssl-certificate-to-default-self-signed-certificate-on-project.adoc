= Resetting {customssl} certificate to default self-signed certificate on {ProjectServer}

[role="_abstract"]
If you want to revert to the default configuration, you can reset a custom SSL certificate to the default self-signed certificate on your {ProjectServer}.

.Procedure
* Reset the {customssl} certificate to default self-signed certificate:
+
[options="nowrap" subs="+quotes,attributes"]
ifdef::containerized[]
----
# {foremanctl} deploy --certificate-source default
----
endif::[]
ifndef::containerized[]
----
# {foreman-installer} --certs-reset
----
endif::[]

.Verification
ifdef::containerized[]
* In your browser, go to `\https://{foreman-example-com}` and inspect the certificate in the browser.
This is typically displayed as a shield, padlock, or tune icon next to the address bar depending on your browser.

* On the command line, display the certificate:
+
[options="nowrap" subs="+quotes,attributes"]
----
# openssl s_client -connect {foreman-example-com}:443 </dev/null
----
endif::[]
ifndef::containerized[]
ifndef::orcharhino[]
* Verify that the following parameters in `/etc/foreman-installer/scenarios.d/{project-context}-answers.yaml` have no values:
endif::[]
ifdef::orcharhino[]
* Verify that the following parameters in `/etc/foreman-installer/scenarios.d/katello-answers.yaml` have no values:
endif::[]

* server_cert:
* server_key:
* server_cert_req:
* server_ca_cert:
endif::[]
