= Updating a disconnected {ProjectServer}

[role="_abstract"]
You can update your air-gapped {Project} to the next patch version by creating a local repository using exported content from a connected {ProjectServer}.

You export synchronized repositories from the connected {ProjectServer}, create local repositories on your air-gapped {ProjectServer}, and update your air-gapped {ProjectServer}.
This process also updates the base operating system on your air-gapped {ProjectServer} when a new minor version is available.

[IMPORTANT]
====
Back up your disconnected {ProjectServer} before updating it.
For more information, see {AdministeringDocURL}backing-up-{project-context}-server-and-{smart-proxy-context}_admin[Backing Up {ProjectServer} and {SmartProxyServer}] in _{AdministeringDocTitle}_.
====
