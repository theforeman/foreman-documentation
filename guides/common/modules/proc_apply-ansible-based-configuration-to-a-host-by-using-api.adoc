= Apply Ansible-based configuration to a host by using {Project} API

[role="_abstract"]
You can apply Ansible-based configuration to your host by using {Project} API.

.Prerequisites
// TODO: Fix permissions
* Your {Project} account has a role that grants the `create_ansible_lifecycle_environment_paths`, `edit_ansible_lifecycle_environment_paths`, `view_ansible_content`, `view_ansible_lifecycle_environment_paths`, `view_ansible_lifecycle_environments`, and `view_organizations` permissions.
* Your host has Ansible roles assigned to it.

.Procedure
. Create a JSON file, for example, `data.json` to define your job invocation:
+
[source, json, options="nowrap", subs="verbatim,quotes,attributes"]
----
{
  "job_invocation": {
    "feature": "ansible_director_configure_host",
    "search_query": "name = _host.example.com_",
    "targeting_type": "static_query"
  }
}
----
. Apply Ansible-based configuration to your host:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--data @data.json \
--header "Accept: application/json" \
--header "Content-Type: application/json" \
--request POST \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/job_invocations/ \
| jq ".id"
----

.Verification
* Query the {Project} API to verify that {Project} shows your newly assigned Ansible roles:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--header "Accept: application/json" \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/job_invocations/_My_Job_ID_/ \
| jq ".status_label"
----
