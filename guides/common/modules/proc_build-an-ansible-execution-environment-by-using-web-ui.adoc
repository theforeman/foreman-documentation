= Build an Ansible Execution Environment by using {ProjectWebUI}

[role="_abstract"]
You can build an Ansible Execution Environment in {Project} by using {ProjectWebUI}.
An Ansible Execution Environment is defined by its base container image and version of `ansible-core`.

.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_execution_environments`, `edit_ansible_execution_environments`, `view_ansible_content`, `view_ansible_execution_environments`, and `view_organizations` permissions.

.Procedure
. In the {ProjectWebUI}, navigate to *Ansible* > *Execution Environments*.
. Click *Create Execution Environment*.
// TODO: Check if field has a name in Web UI
. In the *Name* field, enter the name of your Ansible Execution Environment.
. In the *Base-Image URL* field, enter the name of the base container image.
You can reference any container image from an OCI-compatible container registry or any container image that you have synchronized to {Project}.
+
{Team} recommends to use a fully qualified container image name, for example, `{ansible-director-base-container-image-name}`.
. In the *Ansible version* field, enter the version of `ansible-core`.
+
Ensure that the version is available on the Python Package Index (PyPI), for example, `{ansible-director-ansible-core-version}`.
. Click the *Save* icon to save your Ansible Execution Environment.

.Verification
. In the {ProjectWebUI}, navigate to *Ansible* > *Execution Environments*.
. For your Ansible Execution Environment, verify that the build status icon is green.
