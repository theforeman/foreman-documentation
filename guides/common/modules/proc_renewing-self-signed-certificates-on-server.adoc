= Renewing self-signed certificates on {ProjectServer}

[role="_abstract"]
Renew expired self-signed server and client certificates on {ProjectServer}.
You can also set a custom validity period for the renewed certificates instead of the default 7300 days.

ifndef::containerized[]
[NOTE]
====
To renew the certificate authority (CA) certificate, follow a separate process to avoid disrupting connections to existing hosts.
For more information, see xref:planning-for-self-signed-ca-certificate-renewal[].
====
endif::[]

.Procedure
* Renew server and client certificates:
+
[options="nowrap", subs="+quotes,attributes"]
----
# {foremanctl} deploy --certificate-renew
----
+
To set a custom validity period, append `--certificate-validity-days _My_Days_` with the validity period in days for the new certificate.

.Verification
* In your browser, go to your {ProjectServer} login page, for example, `\https://{foreman-example-com}`, and inspect the certificate in the browser.
This is typically displayed as a shield, padlock, or tune icon next to the address bar depending on your browser.
* On the command line, verify the new certificate validity period:
+
[options="nowrap", subs="+quotes,attributes"]
----
# echo | openssl s_client -connect 127.0.0.1:443 2>/dev/null | openssl x509 -noout -dates
----
+
The output displays the `notBefore` and `notAfter` dates of the certificate, where `notBefore` is the date when the certificate becomes valid and `notAfter` is the date when it expires.

.Next steps
* If you have {SmartProxyServers}, you must update their certificates to trust the renewed {ProjectServer} certificates.
For more information, see {InstallingSmartProxyDocURL}configuring-{smart-proxy-context}-server-with-ssl-certificates_{smart-proxy-context}[Configuring {SmartProxyServer} with SSL certificates] in _{InstallingSmartProxyDocTitle}_.
