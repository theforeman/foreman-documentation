= Import Ansible content from a Git repository by using {ProjectWebUI}

[role="_abstract"]
You can import Ansible content from a Git repository into {Project} by using {ProjectWebUI}.
Select *Git repository* if you want to import Ansible content from a remote Git repository.

.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_content`, `view_ansible_content`, and `view_organizations` permissions.

.Procedure
. In the {ProjectWebUI}, navigate to *Ansible* > *Content*.
. Click *Import Ansible content*.
. Select *Git repository* as source and click *Next*.
. In the *Identifier* field, enter the namespace and name of your Ansible collection as specified in `galaxy.yml` separated by a dot.
. In the *Repository URL* field, enter the URL of your Git repository ending with `.git`.
. If you want to limit the synchronization of Ansible content to specific Git references, enter them in the *References* field.
Click the `+` sign or press enter after each version.
. Click *Add Ansible collection to batch*.
+
Optionally, you can repeat the steps to add another Ansible collection.
. Click *Next*.
. Review your content.
You can click *Delete* to remove an Ansible collection from the batch.
. Click *Finish* to import Ansible content.

.Verification
. In the {ProjectWebUI}, navigate to *Ansible* > *Content*.
. For your imported Ansible content, click the *arrow* symbol to view the list of imported versions.
. Verify that {Project} contains all selected Ansible content and versions.
