= Customize values of Ansible variables for a host by using Hammer CLI

[role="_abstract"]
You can set custom values of Ansible variables for a host by using Hammer CLI.
Setting a value for a specific host overrides all other levels and is helpful to test and debug host configurations.

// TODO: Retest: there is at least one permission missing to actually load the "edit" window
.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_variable_overrides`, `destroy_ansible_variable_overrides`, `destroy_ansible_variables`, `edit_ansible_variable_overrides`, `edit_ansible_variables`, `view_ansible_content`, `view_ansible_variable_overrides`, `view_ansible_variables`, and `view_organizations` permissions.
* Your host has an Ansible environment assigned to it.
* You have installed the Hammer CLI plugin for Ansible Director.
For more information, see xref:install-the-hammer-cli-plugin-for-ansible-director[].

.Procedure
* Set a custom value for an Ansible variable for your host:
+
----
$ hammer
----

.Verification
* Verify that {Project} set the Ansible variable override successfully for your host:
+
----
$ hammer
----

.Next steps
* Apply the configuration to your host.
