= Content access control for {Project} hosts

[role="_abstract"]
{Project} provides a robust set of strategies for controlling what repository content is accessible to your hosts.
You can restrict content access by using core mechanisms, such as content views, lifecycle environments, and content overrides.
Explore how different restriction types, such as those based on architecture, are prioritized when determining content availability.
Discover how activation keys simplify applying these essential access controls during host registration.
