= Content access control for hosts

[role="_abstract"]
You can cap host access to repositories and packages with content view environments, overrides, composite content views, and strict architecture or release rules that you evaluate in a deliberate order.
Use activation keys at registration time and optional operating system or architecture limits on {customrepos} to keep hosts on the intended content.
