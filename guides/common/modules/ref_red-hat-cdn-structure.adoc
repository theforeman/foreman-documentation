= Red{nbsp}Hat CDN structure

[role="_abstract"]
The Red{nbsp}Hat Content Delivery Network (CDN) is a geographically distributed series of static web servers providing Red{nbsp}Hat content and errata.
You can access this content directly through Subscription Manager or {ProjectServer}.

The Red{nbsp}Hat CDN is located at `cdn.redhat.com`.
{Project} and Subscription Manager configure which content is accessible to your hosts based on your subscription entitlements.
The Red{nbsp}Hat CDN is protected by X.509 certificate authentication to ensure that only valid users can access it.

.Directory structure of the Red{nbsp}Hat CDN
====
----
$ tree -d -L 11
└── content
    ├── beta
    │   └── rhel
    │       └── server
    │           └── 7
    │               └── x86_64
    │                   └── sat-tools
    └── dist
        └── rhel
            └── server
                └── 7
                ├── 7.2
                │   └── x86_64
                │       └── kickstart
                └── 7Server
                    └── x86_64
                        └── os
----

* `content` {endash} Root directory for the content.
* `beta` {endash} Directory responsible for the lifecycle of the content.
Common directories include `beta` (for Beta code), `dist` (for Production) and `eus` (For Extended Update Support) directories.
* `rhel` {endash} Directory responsible for the product name.
Usually `rhel` for {RHEL}.
* `server` {endash} Directory responsible for the type of the product.
For {RHEL} this might include `server`, `workstation`, and `computenode` directories.
* `7` {endash} Directory responsible for the release version, such as `7`, `7.2` or `7Server`.
* `x86_64` {endash} Directory responsible for the base architecture, such as `i386` or `x86_64`.
* `sat-tools` {endash} Directory responsible for the repository name, such as `sat-tools`, `kickstart`, `rhscl`.

Some components have additional subdirectories which might vary.
====

This directory structure is also used in Red{nbsp}Hat subscription manifests.
