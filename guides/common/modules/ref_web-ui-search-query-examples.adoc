= Search query examples

[role="_abstract"]
Use these example queries to find specific resources in the {ProjectWebUI}.
Click the search field on any page to see available parameters for that context.

*Basic syntax*

Queries follow the pattern: `parameter operator value`

Text values with spaces must be quoted:

[options="nowrap", subs="+quotes,attributes"]
----
hostgroup = "Web Servers"
----

*Comparison operators*

[options="nowrap", subs="+quotes,attributes"]
----
status.enabled = true                    # Exact match
hostgroup ~ rhel%                        # Case-insensitive pattern match (% = wildcard)
registered_at > 10-January-2025          # Greater than (for dates: "later than")
registered_at <= Yesterday               # Less than or equal to
release_version ^ 8,9                    # In list
has hostgroup                            # Value is present
not has hostgroup                        # Value is absent
----

*Common search patterns*

Find hosts in a specific environment:

[options="nowrap", subs="+quotes,attributes"]
----
environment = production
----

Find hosts registered in the last week:

[options="nowrap", subs="+quotes,attributes"]
----
registered_at > "7 days ago"
----

Find hosts that need errata:

[options="nowrap", subs="+quotes,attributes"]
----
errata_status = errata_needed OR errata_status = security_needed
----

Find hosts with a specific fact value:

[options="nowrap", subs="+quotes,attributes"]
----
facts.architecture = x86_64
----

Find enabled hosts that are not in a failed state:

[options="nowrap", subs="+quotes,attributes"]
----
status.enabled = true NOT status.failed
----

Find hosts matching a name pattern:

[options="nowrap", subs="+quotes,attributes"]
----
name ~ web*prod*
----

*Free text search*

Typing without specifying a parameter searches across multiple fields simultaneously.
For example, typing `192.168` searches host names, IP addresses, MAC addresses, and other fields.

[NOTE]
====
Free text search is slower and less accurate than parameter-based queries.
Use specific parameters whenever possible for better performance and precision.
====
