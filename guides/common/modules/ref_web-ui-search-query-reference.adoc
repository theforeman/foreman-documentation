= {ProjectWebUI} search query reference

[role="_abstract"]
Use search query syntax and operators in the {ProjectWebUI} to filter lists of resources on {ProjectWebUI} pages.
Parameter-based queries help you find the resources you need faster and more precisely than free-text search in large inventories.

Search queries in the page search bar follow the pattern:

[options="nowrap", subs="+quotes,attributes"]
----
parameter operator value
----

To search text with whitespaces, enclose it in quotes:

[options="nowrap", subs="+quotes,attributes"]
----
hostgroup = "Web Servers"
----

Typing without specifying a parameter searches across multiple fields simultaneously, also known as free-text search.
For example, typing `64` in the search bar on the hosts page displays all hosts that have `64` in their name, IP address, MAC address, and architecture.

[NOTE]
====
Free text search is slower and less accurate than parameter-based queries.
Use specific parameters whenever possible for better performance and precision.
====

The search bar supports multiple date and time formats, including the following:

* "10 January 2017"
* "10 Jan 2017"
* 10-January-2017
* 10/January/2017
* "January 10, 2017"
* Today, yesterday, and similar keywords

.Comparison operators accepted by search
[cols="2,2,8",options="header"]
|====
|Operator |Short Name |Description
| = | EQUALS .2+| Accepts numerical, temporal, or text values.
For text, exact case sensitive matches are returned.
| != | NOT EQUALS
| ~ | LIKE .2+| Accepts text or temporal values.
The search is not case-sensitive.
Accepts the following wildcards: _ for a single character, % or * for any number of characters including zero.
If no wildcard is specified, the string is treated as if surrounded by wildcards: %rhel7%
| !~ | NOT LIKE
| &gt; | GREATER THAN .2+| Accepts numerical or temporal values.
For temporal values, the operator &gt; is interpreted as "later than", and &lt; as "earlier than".
Both operators can be combined with EQUALS: &gt;= &lt;=
| &lt; | LESS THAN
| ^ | IN .2+| Compares an expression against a list of values, as in SQL.
Returns matches that contain or not contain the values, respectively.
| !^ | NOT IN
| HAS or set? | {nbsp} .2+| Returns values that are present or not present, respectively.
| NOT HAS or null? | {nbsp}
|====
