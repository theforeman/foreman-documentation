= Assign an Ansible environment to a host by using {Project} API

[role="_abstract"]
You can assign an Ansible environment to your host by using {Project} API.
Ansible environments define the lifecycle of the Ansible content that is used to configure your host.

.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_lifecycle_environment_paths`, `edit_ansible_lifecycle_environment_paths`, `view_ansible_content`, `view_ansible_lifecycle_environment_paths`, `view_ansible_lifecycle_environments`, and `view_organizations` permissions.
* Your {Project} contains an Ansible lifecycle environment with an Ansible Execution Environment and Ansible content.

.Procedure
* Assign an Ansible environment to your host:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--header "Accept: application/json" \
--header "Content-Type: application/json" \
--request POST \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/ansible_director/lifecycle_environments/_My_Ansible_Lifecycle_Environment_ID_/assign/HOST/_My_Host_ID_/ \
| jq
----

.Verification
* Query the {Project} API to verify that {Project} shows your newly assigned Ansible environment:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--header "Accept: application/json" \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/hosts/ \
| jq ".results | .[]" | jq "select(.id == _My_Host_ID_)" | jq ".ansible_lifecycle_environment_id"
----
