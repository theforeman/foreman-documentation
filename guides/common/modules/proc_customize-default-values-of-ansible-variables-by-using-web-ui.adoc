= Customize default values of Ansible variables by using {ProjectWebUI}

[role="_abstract"]
You can set custom default values of Ansible variables in {Project} by using {ProjectWebUI}.
Global custom values help you define a baseline for your environment.

// TODO: Retest: there is at least one permission missing to actually load the "edit" window
.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_variable_overrides`, `destroy_ansible_variable_overrides`, `destroy_ansible_variables`, `edit_ansible_variable_overrides`, `edit_ansible_variables`, `view_ansible_content`, `view_ansible_variable_overrides`, `view_ansible_variables`, and `view_organizations` permissions.
* Your {Project} contains imported Ansible content.

.Procedure
. In the {ProjectWebUI}, navigate to *Ansible* > *Content*.
. From the list of Ansible content, select a version.
. Select an Ansible role.
. Use the toggle to enable the variable override.
. Click the *Edit* icon.
. On the *Base variable* tab, click the *Edit* icon.
. In the *Default value* field, enter your custom default value.
. Click the *Save* icon to submit your change to {Project}.

.Verification
. In the {ProjectWebUI}, navigate to *Ansible* > *Content*.
. From the list of Ansible content, select a version.
. Select an Ansible role.
. Use the toggle to enable the variable override.
. Click the *Edit* icon.
. Verify that {Project} contains your custom default value.
