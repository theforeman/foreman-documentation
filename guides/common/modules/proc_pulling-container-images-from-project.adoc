= Pulling container images from {Project}

[role="_abstract"]
Pull container images from the {Project} container registry by using Podman or Docker to deploy images that you publish in content views across lifecycle environments.

.Prerequisites
* You have reviewed xref:considerations-for-using-the-{project-context}-container-registry[].
* If {ProjectServer} uses a self-signed certificate, you have configured Podman or Docker to trust the {Project} certificate authority.
For more information, see one of the following resources:
** xref:configuring-podman-to-trust-the-certificate-authority[]
** xref:configuring-docker-to-trust-the-certificate-authority[]
* Your {Project} account has the `view_lifecycle_environments`, `view_products`, and `view_content_views` permissions, unless the lifecycle environment allows unauthenticated pull.
* Your {Project} account has the `create_personal_access_tokens` permission.
{Project} automatically generates a token when you run `podman login`.

.Procedure
. Log in to the container registry:
+
[options="nowrap", subs="+quotes,attributes"]
----
$ podman login _{foreman-example-com}_
----
. List container images:
+
[options="nowrap", subs="+quotes,attributes"]
----
$ podman search _{foreman-example-com}_/
----
. Pull a container image:
+
[options="nowrap", subs="+quotes,attributes"]
----
$ podman pull _{foreman-example-com}_/_My_Image_Path_[:__My_Tag__]
----
+
Pull the image by using the same image path schema as the one used to push the image.
The path schema can be either label-based or ID-based.
+
The image tag is optional.
If you do not specify a tag, `podman` pulls the latest tag.
