= Considerations for using the {Project} container registry

[role="_abstract"]
Review naming, push, and pull limitations before you use the {Project} container registry so that container workflows succeed without unexpected errors.

* You cannot push to repositories created by the general repositories endpoints, only to repositories created by the container push workflow.
* You can only push content to {ProjectServer} itself.
If you need pushed content on {SmartProxyServers} as well, use {SmartProxy} syncing.
* The pushed container registry name must contain only lowercase characters.
* Unless pushed repositories are published in a content view version, they do not follow the registry name pattern.
For more information, see xref:configuring-container-registry-name-patterns-for-a-lifecycle-environment[].
This is to ensure that users can push and pull from the same path.
* Users are required to push and pull from the same path.
If you use the label-based schema, pull using labels.
If you use the ID-based schema, pull using IDs.
