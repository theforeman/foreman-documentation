= Promote an Ansible lifecycle environment by using Hammer CLI

[role="_abstract"]
You can promote an Ansible lifecycle environment within an Ansible lifecycle environment path by using Hammer CLI.
Ansible lifecycle environments are named sets of Ansible content and Ansible Execution Environments.

.Prerequisites
// TODO: Fix list of permissions
* Your {Project} account has a role that grants the `create_ansible_lifecycle_environment_paths`, `edit_ansible_lifecycle_environment_paths`, `view_ansible_content`, `view_ansible_lifecycle_environment_paths`, `view_ansible_lifecycle_environments`, and `view_organizations` permissions.
* You have at least two Ansible lifecycle environments within an Ansible lifecycle environment path.
* You have assigned Ansible content, an Ansible Execution Environment, or both, to your left Ansible lifecycle environment.
* You have installed the Hammer CLI plugin for Ansible Director.
For more information, see xref:install-the-hammer-cli-plugin-for-ansible-director[].

.Procedure
. Optional: List all Ansible lifecycle environment paths in {Project}:
+
----
$ hammer
----
. Optional: List all Ansible lifecycle environments with your Ansible lifecycle environment path in {Project}:
+
----
$ hammer
----
. Promote your Ansible lifecycle environment:
+
----
$ hammer
----

.Verification
* Verify that {Project} promoted the Ansible lifecycle environment successfully:
+
----
$ hammer
----
