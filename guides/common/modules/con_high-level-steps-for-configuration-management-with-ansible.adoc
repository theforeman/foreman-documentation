= High-level steps for configuration management with Ansible

[role="_abstract"]
You use Ansible roles to define the desired state of your hosts.
Running the roles enforces the configuration.

Configuration management with Ansible involves the following high-level steps:

. Import Ansible roles into {Project}.
. Assign specific roles to a host or host group.
. Optional: Override Ansible variables per host or host group in the {ProjectWebUI}.
. Run the roles on the host or host group.

As a result, your hosts maintain a consistent configuration over time.
The Ansible roles become a part of their definition.
