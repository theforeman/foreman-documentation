= Create an Ansible lifecycle environment by using {Project} API

[role="_abstract"]
You can create an Ansible lifecycle environment by using {Project} API.
Similar to content management, Ansible lifecycle environments allow you to version control content for different phases of the lifecycle of a host.

.Prerequisites
* Your {Project} account has a role that grants the `create_ansible_lifecycle_environment_paths`, `edit_ansible_lifecycle_environment_paths`, `view_ansible_content`, `view_ansible_lifecycle_environment_paths`, `view_ansible_lifecycle_environments`, and `view_organizations` permissions.
* Your {Project} contains an Ansible lifecycle environment path.
* Your {Project} contains an Ansible Execution Environment.
* Your {Project} contains Ansible content.

.Procedure
. Optional: Get a list of all Ansible lifecycle environment paths:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--header "Accept: application/json" \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/ansible_director/lifecycle_environments/paths/ \
| jq ".results | .[]"
----
. Create a JSON file, for example, `data.json` to define your Ansible lifecycle environment:
+
[source, json, options="nowrap", subs="verbatim,quotes,attributes"]
----
{
  "lifecycle_environment": {
    "name": "_My_Ansible_Lifecycle_Environment_Name_",
    "position": -1
  },
  "lifecycle_environment_path_id": "_My_Ansible_Lifecycle_Environment_Path_ID_",
  "organization_id": "_My_Organization_ID_"
}
----
+
`-1` as position means that you append a lifecycle environment to all lifecycle environments in your path.
. Create an Ansible lifecycle environment:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--data @data.json \
--header "Accept: application/json" \
--header "Content-Type: application/json" \
--request POST \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/ansible_director/lifecycle_environments/ \
| jq
----
. Create a JSON file, for example, `data.json` to define the Ansible Execution Environment of your Ansible lifecycle environment:
+
[source, json, options="nowrap", subs="verbatim,quotes,attributes"]
----
{
  "lifecycle_environment": {
    "description": "_My_Description_",
    "name": "_My_Ansible_Lifecycle_Environment_Name_",
    "execution_environment_id": "_My_Ansible_Execution_Environment_ID_"
  }
}
----
. Assign an Ansible Execution Environment to your Ansible lifecycle environment:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--data @data.json \
--header "Accept: application/json" \
--header "Content-Type: application/json" \
--request PUT \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/ansible_director/lifecycle_environments/_My_Ansible_Lifecycle_Environment_ID_/ \
| jq
----
. Create a JSON file, for example, `data.json` to define the Ansible content of your Ansible lifecycle environment:
+
[source, json, options="nowrap", subs="verbatim,quotes,attributes"]
----
{
  "content_assignments": [
    {
      "id": "_My_Ansible_Content_ID_",
      "version": "_My_Ansible_Content_Version_"
    }
  ],
  "organization_id": "_My_Organization_ID_"
}
----
. Assign Ansible content to your Ansible lifecycle environment:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--data @data.json \
--header "Accept: application/json" \
--header "Content-Type: application/json" \
--request PATCH \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/ansible_director/lifecycle_environments/_My_Ansible_Lifecycle_Environment_ID_/ \
| jq
----

.Verification
* Query the {Project} API to verify that {Project} created the Ansible lifecycle environment successfully:
+
[options="nowrap", subs="verbatim,quotes,attributes"]
----
$ curl \
--header "Accept: application/json" \
--user "_My_User_Name_:__My_Password__" \
https://{foreman-example-com}/api/v2/ansible_director/lifecycle_environments/_My_Ansible_Lifecycle_Environment_ID_/ \
| jq
----
