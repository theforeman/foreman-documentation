= Logging exceptions and limitations

[role="_abstract"]
Some {Project} services have limitations or non-standard behavior for log level configuration.
Understanding these exceptions helps you choose the correct approach when configuring log levels for individual services.

== Services without per-service log level parameters

These services do not have dedicated `{foremanctl} deploy` log level parameters.
To change the log level for these services, use their native configuration mechanisms.

ifdef::satellite,katello,orcharhino[]
* *Candlepin*: Configure Candlepin logging through its native Log4j configuration.
* *Pulp*: Configure Pulp logging through its Django settings.
* *PostgreSQL*: Configure PostgreSQL logging through `postgresql.conf`.
endif::[]
* *httpd*: Configure httpd logging through the `LogLevel` directive in the virtual host configuration.

== Alternative log routing

The Apache HTTP Server (`httpd`) uses `systemd-cat` to pipe log output instead of writing to `stdout` and `httpd` logs use the `-t` (syslog identifier) filter with `journalctl`.
