= Keeping track of your exports

[role="_abstract"]
Query and manage export records on the upstream {ProjectServer} to organize incremental exports and track which content was exported to specific downstream servers.

.Procedure
. Optional: Track the destinations of content exports.
Use the `--destination-server` argument during export to indicate the target server.
This option is available for all `content-export` operations.
+
For example, specify the destination server when exporting a content view version:
+
[options="nowrap" subs="+quotes"]
----
$ hammer content-export complete version \
--content-view="_My_Content_View_Name_" \
--destination-server=_My_Downstream_Server_1_ \
--organization="_My_Organization_" \
--version=1.0
----
. Query export records by listing the exports:
+
[subs="+quotes"]
----
$ hammer content-export list \
--organization="_My_Organization_"
----
