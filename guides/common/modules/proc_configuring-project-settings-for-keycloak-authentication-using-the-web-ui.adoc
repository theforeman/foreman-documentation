= Configuring {Project} settings for {keycloak} authentication using the web UI

If you are configuring a client that will provide {ProjectWebUI} authentication, configure the {Project} client and add {keycloak} as an external authentication source.

.Prerequisites

* For {keycloak-quarkus}: Ensure that the *Client authentication* setting in the {Project} client in the {keycloak-quarkus} web UI is set to `ON`.
See also xref:configuring-the-{project-context}-client-in-keycloak_keycloak-quarkus[].
+
For {keycloak-wildfly}: Ensure that the *Access Type* setting in the {Project} client in the {keycloak-wildfly} web UI is set to *confidential*.
See also xref:configuring-the-{project-context}-client-in-keycloak_keycloak-wildfly[].
* Navigate to the following URL within your realm to obtain values to configure {Project} settings: `\https://{keycloak-example-com}/auth/realms/{Project}_Realm/.well-known/openid-configuration`

.Procedure

In the {ProjectWebUI}:

. Navigate to *Administer* > *Settings*.
. On the *Authentication* tab, configure the following settings:
.. *Authorize login delegation*: Set to `Yes`.
.. *Authorize login delegation auth source user autocreate*: Set to `External`.
.. *Login delegation logout URL*: Set to `\https://{foreman-example-com}/users/extlogout`.
.. *OIDC Algorithm*: Set to `RS256`.
.. *OIDC Audience*: Set to the client ID for {keycloak}.
.. *OIDC Issuer*: Set to `_\https://{keycloak-example-com}/auth/realms/{Project}_Realm_`.
.. *OIDC JWKs URL*: Set to `_\https://{keycloak-example-com}/auth/realms/{Project}_Realm/protocol/openid-connect/certs_`.
. Navigate to *Administer* > *Authentication Sources*.
.. From the *External* menu, select *Edit*.
.. On the *Locations* tab, add the locations that you want to be able to use the {keycloak} authentication source.
.. On the *Organizations* tab, add the organizations that you want to be able to use the {keycloak} authentication source.
.. Click *Submit*.
