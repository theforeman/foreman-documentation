:_mod-docs-content-type: SNIPPET

. Perform the following post-upgrade tasks to ensure your environment is adjusted to any changes applied during the upgrade.
ifeval::["{mode}" == "disconnected"]
.. If you use {insights-iop}, update the Common Vulnerabilities and Exposures (CVE) vulnerability mapping file:
+
* On a connected system, download the latest `cvemap.xml` file from the Red{nbsp}Hat Customer Portal:
+
[options="nowrap", subs="+quotes,verbatim,attributes"]
----
$ curl -o cvemap.xml https://security.access.redhat.com/data/meta/v1/cvemap.xml
----
* Transfer `cvemap.xml` to the disconnected {ProjectServer} you are upgrading, and place it in `/var/lib/foreman/`.
+
{Project} detects the new file and publishes it to the internal {insights-iop} services.
endif::[]
.. Verify whether the default templates changed during the upgrade.
Update any custom templates that you created by cloning a default template to reflect these changes.
+
[NOTE]
====
Use custom provisioning snippets to execute custom code before or after the provisioning process instead of cloning default templates.
You can avoid recreating cloned templates if you use custom provisioning snippets.
For more information about configuring custom provisioning snippets, see {ProvisioningDocURL}running-custom-code-during-host-provisioning-by-using-web-ui[Running custom code during host provisioning by using {ProjectWebUI}] in _{ProvisioningDocTitle}_.
====
.. If your {Project} shows any discovered hosts, reboot them:
... In the {ProjectWebUI}, navigate to *Hosts* > *Discovered hosts*.
... Select *Any Organization* to display all discovered hosts.
... Reboot all discovered hosts.
