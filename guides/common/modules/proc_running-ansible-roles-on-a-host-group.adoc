= Running Ansible roles on a host group

[role="_abstract"]
When a host group has an Ansible role assigned to it, {Project} can run the role on all hosts in the host group at once to apply and enforce the configuration state that the role defines.

.Prerequisites
* The public SSH key from {SmartProxy} is available on the hosts in the host group.
For more information, see {ManagingHostsDocURL}Distributing_SSH_Keys_for_Remote_Execution_managing-hosts[Distributing SSH keys for remote execution] in _{ManagingHostsDocTitle}_.

.Procedure
. In the {ProjectWebUI}, navigate to *Configure* > *Host Groups*.
. From the list in the *Actions* column for the host group, select *Run all Ansible roles*.
. You can view the status of your Ansible job on the *Run Ansible roles* page.
Click *Rerun* to rerun a job.
