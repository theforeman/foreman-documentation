= Associating partition tables with disk encryption

{Project} contains partition tables that encrypt the disk of your host using Clevis/Tang during host provisioning.
You must associate the partition table with your operating system entry.
Then, you assign it to your host group or select it manually during provisioning.

.Procedure
. In the {ProjectWebUI}, navigate to *Hosts* > *Provisioning Setup* > *Operating Systems*.
. Select your {client-os} entry.
. On the *Partition Table* tab, associate the `{client-provisioning-template-type} default encrypted` to your operating system entry.
For more information, see xref:creating-partition-tables_{context}[].

.Next steps
* Create a host group that uses the `{client-provisioning-template-type} default encrypted` partition table.
For more information, see {ManagingHostsDocURL}Creating_a_Host_Group_managing-hosts[Creating a host group] in _{ManagingHostsDocTitle}_.
* If you want to use LUKS to encrypt and decrypt the disks of your host, add the host parameter`disk_enc_passphrase` as type *string* and your cleartext passphrase of the LUKS container as value.
If `disk_enc_passphrase` is not specified, the passphrase defaults to `linux`.
** If you want to use Clevis/Tang to decrypt the disk of your hosts during boot time, add the host parameter `disk_enc_tang_servers` as type *array* and your list of Tang servers (example: `["1.2.3.4"]` or `["{server-example-com}", "5.6.7.8"]`)
+
You do not have to set `disk_enc_passphrase` because the passphrase slot is removed from the LUKS container after provisioning.
