= Configuring certificate validity periods

[role="_abstract"]
By default, {ProjectServer} generates self-signed SSL certificates with a validity period of 7300 days (approximately 20 years).
You can configure custom validity periods for CA certificates and server certificates during installation or later to align with your organization's security policies.

[NOTE]
====
The validity period you specify applies to newly generated certificates.
Existing certificates retain their original validity periods until renewed.
====

.Procedure
* Set certificate validity periods:
+
[options="nowrap", subs="+quotes,attributes"]
----
# {foremanctl} deploy \
--certificate-ca-validity-days _DAYS_ \
--certificate-validity-days _DAYS_
----
+
Replace _DAYS_ with the number of days the new certificates should remain valid.

.Verification
* Verify the certificate validity period:
+
[options="nowrap", subs="+quotes,attributes"]
----
# echo | openssl s_client -connect 127.0.0.1:443 2>/dev/null | openssl x509 -noout -dates
----
+
The output displays the certificate's `notBefore` and `notAfter` dates, where `notBefore` is the date when the certificate becomes valid and `notAfter` is the expiration date.
+
The output displays the certificate's `notBefore` and `notAfter` dates.
