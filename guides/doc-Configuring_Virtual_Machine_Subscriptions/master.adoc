== Introduction

You can use host-based subscriptions for Red{nbsp}Hat Enterprise Linux virtual machines in the following virtualization platforms:

* Red Hat Virtualization
* Red Hat Enterprise Linux Virtualization (KVM)
* Red Hat OpenStack Platform
* VMware vSphere
* Microsoft Hyper-V
* Nutanix AHV

=== Recommendations for subscription usage reporting

For accurate subscription usage reporting, follow these recommendations:

* Configure virt-who properly as described in the following sections of this guide.
* Set up system purpose on your systems and activation keys.
For more information, see the following resources:
** {ContentManagementDocURL}Managing_Activation_Keys_content-management[Managing activation keys] in _{ContentManagementDocTitle}_
** {ManagingHostsDocURL}Editing_the_System_Purpose_of_a_Host_managing-hosts[Editing the system purpose of a host] in _{ManagingHostsDocTitle}_
* In a connected environment, configure the {Project} inventory upload plugin to upload your inventory to {RHCloud} so that you can use the Subscriptions service for subscription usage reporting.
You can configure the plugin in the {ProjectWebUI} by navigating to *Configure* > *RH Cloud* > *Inventory Upload*.
For more information about the Subscriptions service, see {RHDocsBaseURL}subscription_central/1-latest/html/getting_started_with_the_subscriptions_service/index[Getting Started with the Subscriptions Service] in _Subscription Central_.

=== Configuration overview

To allow virtual machines to report host-guest mappings and subscription information through their hypervisors, complete the following steps:

.Prerequisites

* Import a Subscription Manifest that includes a host-based subscription into {ProjectServer}. For more information, see {ContentManagementDocURL}Importing_a_Red_Hat_Subscription_Manifest_into_Server_content-management[Importing a Red Hat Subscription Manifest into {ProjectServer}] in _Managing Content_.
* If you are using Microsoft Hyper-V, enable remote management on the hypervisors.
* If you are using Nutanix AHV, consult link:https://access.redhat.com/articles/6216991[How to configure virt-who for Nutanix AHV to work with RHSM] in the _Red{nbsp}Hat Knowledgebase_.
* Create a user with read-only access and a non-expiring password on each hypervisor or virtualization manager. Virt-who uses this account to retrieve the list of virtual machines to report to {ProjectServer}.
** For Red Hat products and Microsoft Hyper-V, create a virt-who user on each hypervisor that runs Red{nbsp}Hat Enterprise Linux virtual machines.
** For VMware vSphere, create a virt-who user on the vCenter Server. The virt-who user requires at least read-only access to all objects in the vCenter Data Center.

.Procedure overview

. xref:virt-who-configuration-for-each-virtualization-platform_vm-subs-satellite[]. Use the table in this section to plan how to configure and deploy virt-who for your virtualization platform.
. xref:creating-a-virt-who-configuration_vm-subs-satellite[]. Create a virt-who configuration for each hypervisor or virtualization manager.
. xref:deploying-a-virt-who-configuration[]. Deploy the virt-who configurations using the scripts generated by {Project}.
. Register the virtual machines to {ProjectServer}.
For more information, see {ManagingHostsDocURL}registering_hosts_to_server_managing-hosts[Registering hosts] in _{ManagingHostsDocTitle}_.

== Deploying a virt-who configuration

After you create a virt-who configuration, {Project} provides a script to automate the deployment process. The script installs virt-who and creates the individual and global virt-who configuration files.

For Red Hat products, you must deploy each configuration file on the hypervisor specified in the file. For other products, you must deploy the configuration files on {ProjectServer}, {SmartProxyServer}, or a separate Red{nbsp}Hat Enterprise Linux server that is dedicated to running virt-who.

* To deploy the files on a hypervisor or {SmartProxyServer}, see xref:deploying-a-virt-who-configuration-on-a-hypervisor_vm-subs-satellite[].
* To deploy the files on {ProjectServer}, see xref:deploying-a-virt-who-configuration-on-satellite-server_vm-subs-satellite[].
* To deploy the files on a separate Red{nbsp}Hat Enterprise Linux server, see xref:deploying-a-virt-who-configuration-on-a-separate-rhel-server_vm-subs-satellite[].
